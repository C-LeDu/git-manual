git commit -m

Cette commande va enregistrer les modifications préalablement demander dans l'historique et permet de leur associer un nom, ou message descriptif

git commit -m "message descriptif"

git commit -m "exemple"

Affichage

On branch gh-pages
Your branch is up-to-date with 'origin/master'
nothing to commit, working directory clean

git commit -a

Cette commande permet de commiter toutes les modifications qui ont été effectué et permet de leurs associer un nom de commit à partir de la console.

git commit -a

git commit -a

Affichage

" entrer ce qui est affiché dans la console"

git commit --amend

La commande git commit --amend permet de faire des modifications sur les métadonnées du dernier commit si vous avez fait des erreurs dans le commentaire ou si vous avez oublié de add un ficher. ATENTION: Cela change l'histoire (et le SHA) donc n'utilisez pas cette dernière si vous avez déja publié vos commits!

Syntaxe

git commit --amend -m "modification de votre message de commit"

Affichage

$git commit --amend -m "modification de votre message de commit" [gh-pages 512fd5c] modification de votre message de commit 1 file changed, 1 insertion(+),1 deletion(-)
